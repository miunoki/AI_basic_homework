### Linux Ubuntu 系统校内有线网络配置zjunet教程

**使用情况说明：** 因为校内有线网络只能浏览校内网站，无法浏览校外网站，且Linux上没有图形vpn，只能从浙大镜像源下载zjunet，然后登陆账号。

**矛盾点：** zjunet 下载需要依赖 xl2tpd、ppp、iproute2，如果机子上没有预先安装，则需要到各个源上安装，但这就需要连接校外网站，但是校内有线无法连接校外网站，所以在没有zjunet安装完毕的情况下无法下载依赖，如此形成矛盾。

**解决方法：** 修改系统安装源链接，先使用zju镜像源下载依靠和zjunet，之后再调回。

1. 查看操作系统版本
> lsb_release -a

2. 备份镜像源
> 说明：将系统源文件备份到backup里面以便后续出错重建
>
> sudo cp /etc/apt/sources.list （这里有个空格） /etc/apt/sources.list.backup

> 说明：查看源所在的文件夹
>
> sudo ls /etc/apt/

3. 修改源文件
> 编辑主源列表文件，将其内容替换为浙大镜像源
>
> sudo nano /etc/apt/sources.list

> 在打开的 nano 编辑器里，删除全部现有内容，并粘贴以下针对 Ubuntu 22.04 (Jammy) 的浙大镜像配置：
>
> deb https://mirrors.zju.edu.cn/ubuntu/ jammy main restricted universe multiverse
>
> deb https://mirrors.zju.edu.cn/ubuntu/ jammy-updates main restricted universe multiverse
>
> deb https://mirrors.zju.edu.cn/ubuntu/ jammy-backports main restricted universe multiverse
>
> deb https://mirrors.zju.edu.cn/ubuntu/ jammy-security main restricted universe multiverse
>
> 粘贴后，按 Ctrl+O 保存，按 Enter 确认文件名，再按 Ctrl+X 退出编辑器

4. 清理并更新软件源
> 更新软件源列表
> 
> sudo apt-get update

> 清理因之前失败安装产生的无用包
> 
> sudo apt-get autoremove

5. 修复损坏依赖并安装
> 修复损坏的依赖关系
>
> sudo apt --fix-broken install

6. 安装zjunet
> 添加 Zjunet 的软件源和密钥
> 
> curl https://mirrors.zju.edu.cn/openzjunet/zjumirrors.pgp | sudo tee /etc/apt/trusted.gpg.d/zjumirrors.asc
>
> curl https://mirrors.zju.edu.cn/openzjunet/zjunet.list | sudo tee /etc/apt/sources.list.d/zjunet.list

> 更新软件源列表
> 
> sudo apt-get update

> 安装 zjunet
> 
> sudo apt-get install zjunet

7. 配置校园网账号
> 配置浙大统一身份认证账号（学号/工号）和密码
>
> zjunet user add
>
> 运行上面指令后会让你输入账号和密码
>
> 例如：
> ps@ps:~/Desktop$ zjunet user add
>
> username: ...
>
> password: ...

8. 使用vpn
> 连接 VPN
> 
> zjunet vpn -c

> 断开 vpn
>
> zjunet vpn -d

> 查询实时连接状态
>
> zjunet vpn -s

> 查看zjunet基本用法
>
> zjunet usage

> 删除已登录账户（未尝试）
>
> 直接删除配置文件，或者查看 sudo ls -la /etc/zjunet/ 的目录内容
> 
> sudo rm /etc/zjunet/*.conf
>
> 通过命令覆盖配置，重新运行 
>
> zjunet user add
>
> 根据Note: to delete/disable a user, edit /etc/xl2tpd/xl2tpd.conf，需要一个是把 /etc/xl2tpd/xl2tpd.conf 里面自己的条目手动编辑掉，另一个是将 /etc/ppp/peers/ 文件夹下的自己的配置文件删除掉。（来自98前辈的经验发帖）

9. 相关事宜
> 安装（重装升级）xl2tpd时询问Y/I/N/O/D/Z
>
> Y 或 I：安装软件包维护者（Ubuntu官方）提供的新版本默认配置文件。你之前的所有修改将被覆盖
> 
> N 或 O：保留你当前电脑上已存在的配置文件（即旧版本留下的那个）
>
> D：查看两个版本文件的差异
>
> Z：启动一个Shell来检查情况
>
> 选择 N 可以确保之前可能做过的任何有效配置不会丢失。且大概率是默认配置，保留旧文件和新文件在当前几乎没有区别。最重要的是，zjunet 这个工具在运行时，会生成并使用自己的配置文件，不太依赖这个系统级的 /etc/default/xl2tpd 文件。所以这个选择不会影响后续使用 zjunet 连接VPN。

> 关机后重新启动电脑无需再登陆账号，只需在命令行中连接vpn即可
>
> zjunet vpn -c
