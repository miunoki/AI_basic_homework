## 为什么自建加速器？
我们都知道国内网络在访问GitHub官网或拉取Docker镜像时经常会遇到网络不稳定的问题:
* 对于GitHub，要么开梯子花流量，要么使用watt tookit这类工具（它会修改本地host文件、启动一个反代并伪造ssl证书，极易被浏览器识别为不安全并阻止访问），
* 对于Docker，这几年阿里、腾讯等商用的镜像加速器纷纷停止向公众开放，直接使用清华等高校的镜像源或轩辕这种社区镜像是普遍的解决方案，但是仍有不稳定的情况。

此时搭建一个个人的加速器就能同时解决成本和安全性，并带来稳定的访问速度。
## 创建cloudflare worker并写入脚本
* cloudflare worker和传统意义的虚拟主机（云服务器）不同，是一种现代的边缘计算技术（这里不详细说了，我们不需要了解它的原理，只是利用其轻量快速优势）
* 得益于cloudflare免费账户能够创建最多30个 Worker脚本，并拥有10万次/天的请求上限（对个人而言绰绰有余），你只需要有一个cloudflare账户（前往(https://dash.cloudflare.com/)注册并登录，可绑定Google/GitHub账号）

* 对于GitHub加速器，粘贴以下代码（覆盖原来的内容）并保存：
```JavaScript
// 配置需要代理的域名
const GITHUB_DOMAINS = ;

// 代理主函数
export default {
 async fetch(request, env, ctx) {
 const url = new URL(request.url);

 // 处理根路径访问，返回简单说明
 if (url.pathname === "/" && url.search === "") {
 return new Response(`
 <h1>GitHub 代理服务</h1>
 <p>使用方法：在 GitHub 链接前加上本域名即可</p>
 <p>示例：<br>
 - https://${url.host}/github.com/username/repo<br>
 - https://${url.host}/raw.githubusercontent.com/username/repo/branch/file
 </p>
 `, {
 headers: { "Content-Type": "text/html;charset=utf-8" }
 });
 }

 // 解析目标 URL
 let targetUrl = url.pathname.slice(1) + url.search + url.hash;

 // 自动补全协议
 if (!targetUrl.startsWith("http")) {
 // 检查是否是我们要代理的域名
 const isGithubDomain = GITHUB_DOMAINS.some(domain => 
 targetUrl.startsWith(domain)
 );

 targetUrl = isGithubDomain ? `https://${targetUrl}` : `https://github.com/${targetUrl}`;
 }

 try {
 // 创建新的请求
 const newRequest = new Request(targetUrl, {
 method: request.method,
 headers: request.headers,
 body: request.body,
 redirect: "manual"
 });

 // 发送请求并获取响应
 const response = await fetch(newRequest);

 // 处理重定向
 if (response.redirected) {
 const location = response.headers.get("Location");
 if (location && GITHUB_DOMAINS.some(d => location.includes(d))) {
 const newLocation = location.replace(/^https?:\/\//, `${url.protocol}//${url.host}/`);
 return new Response(null, {
 status: response.status,
 headers: { ...response.headers, Location: newLocation }
 });
 }
 }

 // 构造响应，修改部分头信息
 const headers = new Headers(response.headers);
 headers.set("Access-Control-Allow-Origin", "*");

 return new Response(response.body, {
 status: response.status,
 statusText: response.statusText,
 headers: headers
 });
 } catch (e) {
 return new Response(`代理错误: ${e.message}`, { status: 500 });
 }
 }
};
```

## 使用你的专属加速器
* 对于GitHub加速器，直接在GitHub连接前加上你的worker域名即可，如：
```
https://github.username.workers.dev/github.com/username/repo
```
*这里把前缀域名替换为你的真实域名*

使用`git clone`命令时，也可以加上加速器地址：
```bash
git clone https://github.username.workers.dev/github.com/user/repo.git
# 也可以替换远程仓库地址，如：
git remote set-url origin https://github.username.workers.dev/your-username/your-repo.git
```
* 对于docker镜像加速器，可修改`daemon.json`中的镜像源配置：
```json
{
 "registry-mirrors": 
}
# 注意，这里要把docker.username.workers.dev换成你的真实worker域名
```
