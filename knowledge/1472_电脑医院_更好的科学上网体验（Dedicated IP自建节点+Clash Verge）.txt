# 更好的科学上网体验（Dedicated IP自建节点+Clash Verge）

写这贴的时候才发现98上已经有相关的帖子，e.g. (https://www.cc98.org/topic/5386452)。不过也因此lz用的方法和他们略有不同，以下内容均为lz自己的理解，如有问题恳请指正。

### 事情的起因

昨天晚上lz喝完酒回来之后发现自己的Claude账号被封了，但是电脑上还要跑Claude Code，于是写申诉想找回，但过了几天也没收到回信，于是决定自己再去买一个号，结果刚登上去没几分钟又被封了，lz非常恼火，于是大概思考了一下原因。

#### 这里是lz用的第一种科学上网的方式：“万人骑”机场/梯子+Clash Verge

Lz觉得主要是因为梯子不是很好，里面的代理节点的IP不干净。

> **如何确定自己的IP干不干净呢？**可以进https://ping0.cc/网站测试，结果示例如图：
>
> <img src="https://file.cc98.org/v4-upload/d/2026/0222/cxmrsbps.webp" alt="img" style="zoom:60%;" />
>
> 风控值和共享人数越低越好，根据Claude账号卖家所说，风控值低于25%比较好。
>
> lz的节点测出来最低的38%，最高的有100%

以前我们总觉得有个梯子，能接得上GitHub、谷歌学术等等就行，很多时候也会贪便宜，但这样会出现节点太脏，一个IP被数千人共用，如果有一个人乱搞，整个IP就会被别的公司标记、拉黑。
或许也有一些梯子的节点人数不多，可以找个新的梯子用，但lz认为这样治标不治本，等到用的人多起来了还是会出问题。后来尝试了第二个方法。

### VPN (Virtual Private Network)

现在看来这不是一个好的选择，不过探索的时候不了解，后面才慢慢发现。
我试用了ExpressVPN和Proton VPN两款产品。直观上问题很明显

1. 全局代理，**过于死板**，所有的流量全部经过代理节点，发个微信、看个B站数据都要先去国外逛一圈。对原本中国访问起来比较快的应用不好，速度慢
2. **速度慢**，VPN产品专注于数据的安全与隐私，加密过程需要一些时间，所以访问很多网站都会稍慢。

**传统的VPN**设计初衷就是为了“安全与隐私”，而不是为了翻墙，最早是给跨国企业用的，方便员工在家里链接企业内网，防止黑客。所以它更注重“加密”而不注重“伪装”，若是差一点的VPN也很容易被防火墙（GFW）识别并掐断。

而我们所说的“**机场**”，也就是**代理节点**，就是专门为了欺骗防火墙而生的，不仅加密，更注重**伪装**。

1. 代理协议会将我们的翻墙流量伪装成正常的访问https网站的流量，骗过防火墙。
2. 配合Clash Verge这样的客户端，可以做到**分流**，以及自定义规则等，让国内流量直接连，国外流量走代理，十分灵活。

现在两种方法都有问题，VPN的IP很安全，但没法用Clash Verge分流，速度慢；而机场的IP被污染，容易导致被封号。

### 解决方法

于是我们的答案呼之欲出：**Clash Verge 分流 + 专属的（Dedicated）IP**，往往用的是静态住宅IP，这样就像是一个美国本地人在访问网站。而机房IP是被大家深恶痛绝的，我们要避免被识别为这个。
我们可以去想办法购买到一个安全的IP地址，配合Clash Verge将国外的代理节点设置成它，这样就完成任务
这个时候任务已经彻底超出lz的知识范围了，于是我们请教了Gemini老师。它给出了两种方案：

<img src="https://file.cc98.org/v4-upload/d/2026/0222/e2erc5q5.webp" alt="Gemini_Generated_Image_rql7alrql7alrql7" style="zoom:80%;" />
如果你两种方式都去搜索一下会发现，后面直接买静态住宅IP的服务便宜很多，大概每个月2美元左右，但为什么推荐第一种呢？

1. 国内端：租了 VPS，就相当于拥有一台完整的服务器。你可以安装 **VLESS-Reality** 这种目前最安全的加密协议。它能把你的代理流量伪装成正常的网页浏览，GFW 识别不出来。

 第二种很多服务只提供（SOCKS5 or HTTP）的代理，不做伪装，容易被防火墙封锁掉。

2. 国内-国外连接：像丽萨主机、搬瓦工、DMIT 很多都带有 CN2 GIA 或 AS9929 这种“中美专线”优化，速度很快。

 第二种一般用的是“standard, cheap commercial routing”，~~我有点不知道怎么翻译~~，和国外的信息传输是Routing。如果比喻成公路的话，那么其中有一些就是**VIP高速公路**，一些就是**拥挤的国道**。

3. 国外端：最后“Double ISP"意味着IP的属性相当于家庭宽带，而不是机房IP，满足了我们的核心要求。

> VPS (Virtual Private Server) 和 Double ISP 并没有那么简单，感兴趣可以深入了解。

那么接下来确认了目标，可以开始实操了。

### 实际操作

1. 我选择的提供商是(https://lisahost.com)。(可以尝试一下别的，因为这家公司的IP好像被谷歌标记了，后面会讲解决方法)
 <img src="https://file.cc98.org/v4-upload/d/2026/0222/vtmdcjs1.webp" alt="image-20260221223115430" style="zoom:50%">

 - 我们选择第一个服务，有双ISP，并且有VIP高速公路。
 <img src="https://file.cc98.org/v4-upload/d/2026/0222/itybhxol.webp" alt="image-20260221223330982" style="zoom:50%;" />

 - 然后滑到最下面可以购买，我选择的是一年的套餐，499元。
 - 买完后来到左边栏的“产品服务”，选择“我的产品”，点击一大串加粗的“产品的名字”，就是**美国顶级…………特价年付版**
 <img src="https://file.cc98.org/v4-upload/d/2026/0222/xrqs1vxw.webp" alt="image-20260221223657682" style="zoom:50%;" />
 - 进去后可以看到IP地址，端口，用户名，密码等信息。
 - 买到IP之后可以去https://scamalytics.com这个网站测试一下。

2. 接着打开电脑的终端
 输入`ssh -p port root@ip`,==注意将port这四个字母替换为端口，ip替换为ip地址==，
 例如`ssh -p 12345 root@127.0.0.1`

 运行后输入密码。

3. 接下来运行：
 `bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)`

 安装3x-ui Proxy Panel，可以打开图形化界面，方便之后代理的配置。

 **路上会让你选一个端口号，不要选到什么80，443这种端口就行，选一个稳妥的，比如52034？**

 过程中会遇到一些选择：

 - <img src="https://file.cc98.org/v4-upload/d/2026/0222/b4sgdgp0.webp" alt="image" style="zoom:50%;" />

 选择2，按回车，

 - 接下来一路按回车就行：

 - Using Let's Encrypt for IP certificate (shortlived profile)...
 Do you have an IPv6 address to include? (leave empty to skip):
 - Port to use for ACME HTTP-01 listener (default 80):

4. 这一步结束后会给你一个URL，（https;//ip:port/...）和用户名密码，你在浏览器里打开，输入用户名密码进入就可以看到一个控制台界面
 <img src="https://file.cc98.org/v4-upload/d/2026/0222/xngkdvs3.webp" alt="image-20260221230405537" style="zoom:50%;" />

5. 接下来我们就可以开始把这个VPS配置为我们的代理节点了

 点击侧边栏的inbounds，选择Add Inbound。
 Protocol选择`vless`， Port选择`443`， Network选择`tcp`，Security选择`Reality`，在下面新展开的按钮"Get New Keys"中自动创建密钥。其他的不用动，点击""Create"完成创建。

 接下来点击这个inbound最左边的加号可以展开信息：
 <img src="https://file.cc98.org/v4-upload/d/2026/0222/13uys3pw.webp" alt="image-20260221231042548" style="zoom:33%;" /><img src="https://file.cc98.org/v4-upload/d/2026/0222/k5ck3vft.webp" alt="image-20260221231121285" style="zoom:23%;" />

 点击感叹号，滑到最下面的到一个vless开头的url一样的东西，我们复制它。

6. 接下来我们去到(https://sub.v1.mk/)，将复制的内容填入“订阅链接”，然后点击下方的“生成订阅链接”，在“定制订阅”中获得我们要用到Clash Verge里面的URL。

 接下来我们进入Clash Verge侧边栏的`Profiles`

 <img src="https://file.cc98.org/v4-upload/d/2026/0222/pszsdzad.webp" alt="Screenshot 2026-02-21 at 23.34.08" style="zoom:50%;" />

 然后点击右上角的New，只需要在Name填一个名字，然后把之前生成的URL输入到`Subscription URL`里面即可，最后再点一下新生成的标签块，背景边框会变蓝，就完事了，正常来说这时候就已经完成了配置。

### 一些问题

但是由于我们选择的IP也没有很好，或许是因为这个公司的IP被谷歌标记了，不知道具体什么原因，我做了各种测试，只有谷歌的服务登不了。所以可以在选择Double IPS时可以尝试使用别的服务。

问Gemini，给出的方式是在我们的VPS服务器上安装**Cloudflare WARP**，"WARP will give your VPS a secondary, 100% clean US IP address."。我们将和谷歌相关的流量导到另一个IP地址上去。避免这个公司的IP被标记的问题。

接下来的步骤我具体也不是很了解：

1. **Install the Official Cloudflare Client**
 在VPS上运行：

 ````bash
 curl -fsSL https://pkg.cloudflareclient.com/pubkey.gpg | sudo gpg --yes --dearmor --output /usr/share/keyrings/cloudflare-warp-archive-keyring.gpg
 echo "deb https://pkg.cloudflareclient.com/ jammy main" | sudo tee /etc/apt/sources.list.d/cloudflare-client.list
 sudo apt-get update && sudo apt-get install cloudflare-warp -y
 ````

2. **Turn on the Proxy**

 ````bash
 warp-cli registration new
 warp-cli mode proxy
 warp-cli proxy port 40000
 warp-cli connect
 ````

 依次运行这四行命令。

3. 回到之前的控制台页面：https://ip:port/...

 点击`Xray COnfigs -> Outbounds -> Add Outbound`

 <img src="https://file.cc98.org/v4-upload/d/2026/0222/suvuncsi.webp" alt="image-20260221235133358" style="zoom:30%;" />

 配置成这样后确认。其中`Tag`就是起个名字，方便之后调用，可以自己取。

4. 然后点击`Outbounds`左边的`Routing Rules -> Add Rule`

 <img src="https://file.cc98.org/v4-upload/d/2026/0222/vxv2afcu.webp" alt="image-20260221235331400" style="zoom:30%;" />

 这里Domain指的是作用的域名，比如我们可以写：
 `geosite:google,domain:gemini.google.com,domain:aistudio.google.com`

 关键在于第一条`geosite:google`，标记了谷歌服务。

 然后`Outbound Tag`选择刚才我们起了名字的那个。

5. 最后在页面的顶端点击`Save`，然后`Restart Xray`，这时候再登录谷歌服务就没问题了。

之后遇到类似的问题，我们都可以如法炮制，其实只要更换`Domain`里面的内容就可以了。
