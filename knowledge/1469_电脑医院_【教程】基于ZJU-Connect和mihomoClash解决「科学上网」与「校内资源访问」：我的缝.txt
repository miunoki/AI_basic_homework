## 前言

假期里，笔者有时需要使用代理访问Github、Gemini等网站，同时也需要连接校内服务器和水CC98（）。对于后者，老版RVPN客户端本就十分难用，新版aTrust更是没好到哪去；它们强制接管所有流量，让你频繁切换状态；可能污染DNS；深信服的后台服务会扫描你的外接设备导致硬盘/U盘无法弹出。
同时，笔者在计算机网络方面也是小白，平时遇见网络问题代理问题只能干瞪眼或者问AI 也想借此机会增进理解，不至于一问三不知因此也希望各位前辈们多多指教，讨论更好的解决方案呀~
站内的方案多是基于JavaScript脚本覆写为校内流量增加新的规则，但笔者最近刚换到Linux系统，使用的是这个代理环境(https://github.com/nelvko/clash-for-linux-install)，似乎不支持脚本覆写；后来我看见这一篇( (https://www.cc98.org/topic/6384563) )，从而萌生了直接基于配置文件做改动的想法。
但是在拖延症的帮助下，直到春节完了我才开始尝试，本文草稿更是在高铁上写就的，现在就在寝室里倒也是用不上这套方案了。

# Chap1: clash的安装及机场获取
详见本版其他帖子()
linux用户我在此安利一下这个(https://github.com/nelvko/clash-for-linux-install), 至少我用起来很舒服, 只需执行以下命令
```shell
git clone --branch master --depth 1 https://gh-proxy.org/https://github.com/nelvko/clash-for-linux-install.git \
 && cd clash-for-linux-install \
 && bash install.sh
```
就可以解决最基础访问Github的问题
## Chap2: zjuconnect的安装和配置
### linux/mac
建议在(https://github.com/Mythologyli/zju-connect/releases)根据机型获取beta或是NightlyRelease版本
参考(https://www.cc98.org/topic/6372895)中配置文件如下
```yaml
username = "你的学号"
password = '你的VPN密码'
protocol = "atrust"
# graph_code_file = "CheckCode.jpg" 注释此行, 因现已支持web点击验证码
client_data_file = "clientData.json"

# Advanced settings
server_address = "vpn.zju.edu.cn"
server_port = 443
disable_server_config = false
disable_zju_config = false
disable_zju_dns = false
disable_multi_line = false
proxy_all = false
socks_bind = ":21080"
socks_user = ""
socks_passwd = ""
http_bind = ":28888"
shadowsocks_url = ""
dial_direct_proxy = ""
tun_mode = false
add_route = false
dns_ttl = 3600
disable_keep_alive = false
zju_dns_server = "10.10.0.21"
secondary_dns_server = "114.114.114.114"
dns_server_bind = ""
dns_hijack = false
debug_dump = false

port_forwarding = []
custom_dns = []
custom_proxy_domain = []
```
注意socks5绑定端口为21080, 如更改此处, 后续也需一同更改
运行命令
```shell
./zju-connect -config ./config.toml
```
会弹出窗口进行验证码认证，随后需要通过手机收取验证码二次认证, 认证后会生成`client_data.json`以便下次自动复用

此处可以通过systemd注册为服务使其开机自启, 与(https://www.cc98.org/topic/6372895)中所述基本一致, 但我也踩了几个小坑
建议提前将原软件(zju-connect)、配置(config.toml)和缓存文件(client_data.json)移动到系统文件夹下, 我移动到了`/opt/zjuconnect`下
使用命令
```
sudo nano /etc/systemd/system/zju-connect.service
```
创建配置文件,并粘贴以下内容：
```

Description=ZJU Connect VPN Client
After=network-online.target
Wants=network-online.target

Type=simple
User=你的用户名
WorkingDirectory=/opt/zjuconnect # 需要对应更改
ExecStart=/opt/zjuconnect/zju-connect -config /opt/zjuconnect/config.toml

Restart=on-failure
RestartSec=10s

StandardOutput=journal
StandardError=journal
SyslogIdentifier=zju-connect
NoNewPrivileges=true
PrivateTmp=true

WantedBy=multi-user.target
```
注册并启动服务，查看服务的启动情况
```
# 重新加载 systemd 配置
sudo systemctl daemon-reload
# 启用开机自启动
sudo systemctl enable zju-connect.service
# 立即启动服务
sudo systemctl start zju-connect.service
# 查看服务状态
sudo systemctl status zju-connect.service
```
笔者没有mac无法自己尝试配置, 具体方法请参照(https://www.cc98.org/topic/6372895)
最后可以尝试在未开启其他代理的情况下链接CC98
```
curl -I https://www.cc98.org
```
### windows
可以按照Linux的思路来配置，但是如何开机自启笔者未曾自行探索；
在此笔者使用gui版本(https://github.com/mythologyli/zju-connect-for-Windows)亦可以完成配置，笔者选用的是Nightly Release版本，不确定其他版本是否能胜任
安装后按照左上角文件->设置->通用的路径输入账号密码，选择开机自启和启动后自动连接,切换到RVPN选项卡, 选择aTrust模式, 取消勾选DNS劫持(TUN)和自动设置系统代理系统代理；如果你想照抄我的配置, 需要修改socks5端口到21080, 和上面一致, 随后应用并开始代理, 经过登录后应该就能以此命令尝试在未开启其他代理的情况下链接CC98
```
curl -I https://www.cc98.org
```
