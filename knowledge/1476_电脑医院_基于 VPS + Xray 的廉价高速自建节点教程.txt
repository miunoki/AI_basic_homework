### 0. 预备知识

**VPS**

VPS（Virtual Private Server，虚拟专用服务器）是一种通过虚拟化技术将一台物理服务器分割成多个独立虚拟服务器的服务。简单来说，就是一台配置极低的云服务器，但是具有极低的流量成本。

**Xray**

Xray 是一种 V2Ray 内核。简单来说，就是一个没有 GUI 客户端。

**Reality 协议——THE NEXT FUTURE**

 基于“偷证书”的流量加密技术，当我们直接通过防火墙时，具有相对于 Shadowsocks、Trojan、VMess 更强的伪装性。

### 1. VPS 购买

### 3. VPS 环境初始化

现在服务器里面是一个全新的系统。第一步当然是 `apt upgrade` 和 `apt update`（对于 Debian 10，因为其生命周期已经结束，因此需要修改 apt 源为 archive.debian.org ）

然后为了安全性（我们暴露在公网环境，是极度危险的），我们需要禁用 root 账户的 ssh 登录，并修改 ssh 端口。我一般直接禁用密码登录，只使用密钥登录。为了不让教程太冗长，这部分就略去。最后，我们应当创建了一个拥有 sudo 权限的新账户。

### 4. VPS 防火墙配置

首先安装 `ufw` （对于 `CentOS` 是 `firewalld`，当然直接用 `iptables` 也没什么问题 XD）
```
sudo apt update
sudo apt install ufw
```
然后千万不要直接 `sudo ufw enable`！因为这会直接断掉我们的 ssh 连接（ufw 默认拒绝所有传入连接，允许所有传出连接）。（补救措施是通过 VPS 服务商后台的 VNC 登录）
首先开放 ssh 的端口（比如在 53535，看上一步怎么改的）
```
sudo ufw allow 53535
```
然后我们就可以 
```
sudo ufw enable
sudo ufw status
```
来看看它是不是正常工作了
下面我们需要放行 80, 443 端口——这是为了伪装成正规网站，为下一步做准备。
```
sudo ufw allow 80
sudo ufw allow 443
```
我推荐同时放行 `TCP/UDP`，这样我们可以实现更多的协议（如 Hysteria 2）

### 6. Xray 配置

这是非常复杂的一步。我在这里只讲怎么配置一个 Reality+Vless+Vision+XTLS 协议的服务。感兴趣的同学可以看官方文档：（虽然也不全就是了，也可以去看 Gituhb 的讨论区）

对于科学上网，核心问题是怎么把不合理的越境流量伪装成正常流量。为了做到这一点，Reality 协议使用了一个极为天才的想法——偷正规网站的证书给自己的 TLS 流量加密——如此在外界看来便是非常正常的 TLS over TCP 流量了，也就避免了传统的 TLS in TLS 流量指纹。

那么什么网站的证书适合偷呢？有以下几条原则：

1. 使用 TLS 1.3 协议
2. 使用 X25519 签名算法（MLKEM768 抗量子加密算法不影响，但如果证书长度超过 3000 字符，则可以启用 Reality 的后量子加密）
3. 支持 HTTP/2 协议（H2）
4. 不使用 CDN - 如果 Reality 目标网站使用 CDN，数据将转发到 CDN 节点，使你的 Reality 节点成为别人的反向代理加速节点
5. 中国境内不依赖任何代理可以直接访问

符合这些规则的，一般是 VPS 服务器当地的大学官网、旅游局、政府官网等等，也可以偷自己的域名（有机会再讲）。

对于候选网站，我们按 F12 打开开发者工具。在“隐私与安全”下将显示：

![](https://file.cc98.org/v4-upload/d/2026/0109/03vqalea.webp)

这符合要求

然后检验 http：在控制台输入 `window.chrome.loadTimes()?.npnNegotiatedProtocol`
