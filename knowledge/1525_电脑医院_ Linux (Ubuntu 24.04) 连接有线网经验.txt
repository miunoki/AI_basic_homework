# Ubuntu 24.04 连接 ZJU 有线网络经验

说明：本文部分内容基于 AI 辅助整理生成，命令和关键判断已结合本人实际排查过程核对；网络环境和设备差异较大，仅作为参考。文末也附上了我参考过的前辈经验贴，建议遇到具体报错时一起对照阅读。

（26年5月补充）学校部分地方的网络认证似乎变成了网页认证，net3.zju.edu.cn / net.zju.edu.cn

最近给一台新电脑装 `Ubuntu 24.04.4 LTS`，接 ZJU 有线网络时踩了一些坑，整理一下过程，既作为留档，也希望能给遇到类似问题的同学提供参考。

我的情况是：网线插上以后机器能接入校园网，也能拿到类似 `10.x.x.x` 的地址，但不能正常访问外网。学校官方没有可直接使用的 Linux 认证客户端，尝试 `zju-connect` 也没有连上，询问信息技术中心后得到的反馈是需要自行解决。最后可用的方案是使用求是潮维护的 `zjunet`。

关键工具：

- `zjunet`：求是潮产品研发中心（QSCTech）维护的 ZJU 网络命令行工具。
- (https://mirrors.zju.edu.cn/openzjunet/)：主要提供安装方式、依赖说明和常见问题，不算完整命令手册；安装后建议用 `zjunet usage` 查看命令。
- (https://mirrors.zju.edu.cn/openzjunet/zjunet.list)

本文只针对 Ubuntu 24.04 下的普通桌面/工作站使用场景。Windows、macOS、路由器、OpenWrt 的配置不在这里展开。

## 我的环境

- 系统：`Ubuntu 24.04.4 LTS`
- 主板：`MAG B850 TOMAHAWK MAX WIFI`
- 场景：Windows 改装 Linux，接有线网络

## 双系统或 Windows 快速启动导致网卡识别异常

如果是双系统、曾经安装过 Windows、无法确认是否关闭过快速启动，Linux 安装完成后重启时可能出现有线/无线网卡都识别不到的情况。

这不一定是 Linux 驱动缺失，也可能是 Windows 快速启动或残留供电影响了网卡初始化，导致 Linux 启动后没有把网卡正常拉起来。

可以尝试：

1. 完全关机。
2. 拔掉电源线。
3. 拔掉网线。
4. 等待几分钟，让主板和网卡充分放电。
5. 重新接上电源和网线，再启动 Linux。

如果这样恢复后网卡能出现，再继续按 `nmcli` 和 `zjunet` 的流程排查。如果仍然完全识别不到网卡，再考虑 BIOS 设置、内核版本、网卡驱动和硬件故障。

## 快速判断是不是同一个问题

先运行：

```bash
lsb_release -a
nmcli device status
nmcli general status
```

如果 `nmcli device status` 能看到有线网卡，并且状态不是完全 `disconnected`，继续看有线网口有没有拿到 IP：

```bash
nmcli connection show
```

如果已有可用连接，例如 `配置 2`：

```bash
nmcli connection up '配置 2'
```

再查看地址。可以直接用：

```bash
ip addr
```

如果能看到类似：

```text
10.xx.xx.xx/22
```

说明有线已经接入校园网。此时如果不能访问外网，更可能是认证没完成，而不是网线没插好。

## 安装 zjunet 依赖

`zjunet` 需要一些依赖：

```bash
sudo apt update
sudo apt install xl2tpd curl bind9-dnsutils
```

这一步不一定需要梯子，但需要当前 apt 源可访问。推荐顺序是：

1. 如果手机热点、USB 共享网络或校网 Wi-Fi 能上网，先用这些网络安装依赖。
2. 如果所有网络都不方便，但当前 Ubuntu 源不可达，可以先换成国内镜像源。
3. 换源时必须按实际系统版本选择发行版代号，不要跨版本混用。

确认发行版代号：

```bash
lsb_release -cs
```

或：

```bash
. /etc/os-release
echo $VERSION_CODENAME
```

常见对应关系：

```text
Ubuntu 24.04 -> noble
Ubuntu 22.04 -> jammy
Ubuntu 20.04 -> focal
```

依赖安装后可以简单检查：

```bash
xl2tpd -v
curl -V
dig -v
```

如果 `sudo apt update` 部分失败，先看失败的是哪个源。如果只是一些第三方外网源失败，而 Ubuntu 基础源或浙大镜像源还能正常更新，可以继续尝试安装依赖；如果所有源都不通，先确认网络状态、DNS 和是否能访问 `mirrors.zju.edu.cn`。

## 添加 zjunet 仓库

这里不需要改 Ubuntu 主源，只添加 `zjunet` 仓库。

```bash
curl https://mirrors.zju.edu.cn/openzjunet/zjumirrors.pgp | sudo tee /etc/apt/trusted.gpg.d/zjumirrors.asc >/dev/null
curl https://mirrors.zju.edu.cn/openzjunet/zjunet.list | sudo tee /etc/apt/sources.list.d/zjunet.list
```

检查一下：

```bash
cat /etc/apt/sources.list.d/zjunet.list
```

理想输出：

```text
deb https://openzjunet.pages.zjusct.io/debian stable main
```

关于浙大镜像站能不能在校外访问，建议保守理解：`openzjunet` 相关路径的优势是校内也能访问，适合还没有完成外网认证时安装 `zjunet`；整个 `mirrors.zju.edu.cn` 的不同镜像项目可能有不同访问策略，不要假设所有路径在任何网络下都稳定可用。

## 安装 zjunet

```bash
sudo apt update
sudo apt install zjunet
```

如果没有梯子，`sudo apt update` 里其他外网源可能失败或被忽略。一般来说，只要 `zjunet` 仓库和必要的 Ubuntu 依赖源能成功读取，就可以继续安装；失败的第三方源可以等连上外网后再处理。也可以先把 Ubuntu 主源换成当前系统版本对应的国内镜像源。

安装后检查：

```bash
zjunet version
zjunet usage
```

## 配置账号并连接

排查时建议先断开 Wi-Fi、手机共享和代理连接，只保留校园有线。

添加统一认证账号：

```bash
zjunet user add
```

按提示输入学号/工号和统一认证密码。不要把真实账号密码写进公开帖子或脚本里。

如果当前开着 `ZJUWLAN` 或代理连接，例如 `Mihomo`，可以先断开：

```bash
nmcli connection down ZJUWLAN
nmcli connection down Mihomo
```

连接 VPN：

```bash
zjunet vpn -c
```

查看状态：

```bash
zjunet vpn -s
```

断开：

```bash
zjunet vpn -d
```

如果 `zjunet vpn -s` 显示已连接，但浏览器仍然不能访问外网，优先检查默认路由、DNS、Wi-Fi 或代理是否还在影响网络。

## 我这次实际用到的最短流程

```bash
sudo apt update
sudo apt install xl2tpd curl bind9-dnsutils
curl https://mirrors.zju.edu.cn/openzjunet/zjumirrors.pgp | sudo tee /etc/apt/trusted.gpg.d/zjumirrors.asc >/dev/null
curl https://mirrors.zju.edu.cn/openzjunet/zjunet.list | sudo tee /etc/apt/sources.list.d/zjunet.list
sudo apt update
sudo apt install zjunet
zjunet user add
nmcli connection down ZJUWLAN
nmcli connection down Mihomo
zjunet vpn -c
zjunet vpn -s
```

## 总结

1. 有线网卡能拿到 `10.x.x.x` 地址，不代表已经能访问外网。
2. Ubuntu 24.04 可以换国内镜像源，但必须使用当前系统对应的发行版代号。
3. `sudo apt update` 不一定需要梯子，但需要当前 apt 源可访问；如果还没完成校园网认证，部分外网源失败是常见情况。
4. 如果手机热点、USB 共享网络或校网 Wi-Fi 能上网，可以先用它们安装依赖和 `zjunet`，再切回有线网络做认证。
5. 排查 `zjunet` 认证时，最好先断开 Wi-Fi、手机共享、代理和虚拟网卡，只保留校园有线，避免默认路由被干扰。
