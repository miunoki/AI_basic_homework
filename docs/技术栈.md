# 技术栈

本文档记录项目当前使用的主要技术、关键实现方案和技术变更记录。后续凡是涉及检索、生成、认证、界面、部署、数据处理等技术方案的修改，都需要同步更新本文档。

## 维护规则

- 新增技术能力时，在“当前技术栈”中补充对应条目。
- 修改已有技术方案时，更新对应说明，并在“技术变更记录”中追加一条记录。
- 废弃或替换技术时，说明替换原因和新方案。
- 只记录技术层面的变化；普通文案、截图、报告排版等非技术修改不需要记录。

## 当前技术栈

| 模块 | 技术/方案 | 说明 |
|------|-----------|------|
| 编程语言 | Python | 项目主体逻辑、命令行入口、网页入口、检索与实验脚本均使用 Python 实现。 |
| 大模型调用 | DeepSeek API + OpenAI Python SDK | `llm.py` 使用 OpenAI SDK 兼容接口调用 `deepseek-chat`，支持从 `config.py` 或环境变量读取 API Key。 |
| RAG 架构 | 检索增强生成 | 用户提问后先从本地 CC98 知识库检索相关帖子，再将参考内容拼接进 Prompt，由大模型生成回答。 |
| 知识库 | 本地文本文件 | `knowledge/` 下每个 `.txt` 文件作为一个知识条目，内容来自 CC98 论坛帖子。 |
| 关键词检索 | 中文 2-gram + Jaccard + 标题加权 | 默认检索模式，无需额外模型，启动稳定；标题命中会获得额外权重。 |
| 语义检索 | sentence-transformers | 可选进阶模式，使用多语言向量模型做语义相似度检索，并支持 embedding 缓存和失败回退。 |
| 上下文增强检索 | 短追问拼接上一轮问题 | `main.py` 中当用户输入较短追问时，将上一轮用户问题拼接进检索 query，提高多轮追问命中率。 |
| Prompt 构造 | 参考帖子 + 用户问题 + 来源提示 | 将 top 相关帖子截断后拼入 Prompt，并要求模型基于资料回答、在末尾标注来源。 |
| 命令行界面 | 交互式 CLI | `main.py` 提供终端问答入口，支持多轮历史上下文。 |
| 网页界面 | Gradio Blocks | `app.py` 提供浏览器聊天界面、示例问题、访问控制和自动打开浏览器。 |
| 网页访问控制 | 访问码 / 个人 API Key 双通道 | 新生可输入统一访问码使用服务端 API；已有个人 DeepSeek API Key 的用户可直接输入 Key，无需访问码。 |
| 配置管理 | `config.py` + 环境变量 | 服务端 API Key 和访问码可通过 `config.py` 或 `DEEPSEEK_API_KEY`、`ACCESS_CODE` 环境变量配置。 |
| 启动方式 | `start_web.bat` | Windows 下可双击启动网页服务，降低非开发者使用门槛。 |
| 对比实验 | RAG vs 无知识库 | `compare.py` 使用固定测试问题对比带知识库和不带知识库的回答效果。 |
| 依赖管理 | `requirements.txt` | 记录 OpenAI SDK、sentence-transformers、Gradio、requests 等依赖。 |
