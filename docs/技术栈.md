## 三、RAG 技术路线

### 3.1 检索与范围判断

默认模式为 `keyword`，流程为：

1. 判断问题是否属于浙江大学校园生活范围；
2. 扩展地点词和主题同义词；
3. 进行中文 2-gram 分词并过滤泛化问法；
4. 结合具体标题词、原始标题词、扩展标题词、主题命中、Jaccard 和短语命中进行加权；
5. 过滤低于 50 分的弱相关结果，召回 Top-8。

可选的 `semantic` 模式使用 `paraphrase-multilingual-MiniLM-L12-v2` 生成向量，以余弦相似度检索。向量缓存记录模型名和知识库指纹；模型初始化或下载失败时自动回退到关键词模式，避免整个应用不可用。

### 3.2 Prompt 与来源

- 检索召回 Top-8，Prompt 只使用前 4 条；
- 每条参考资料最多保留 900 字符；
- System Prompt 要求模型仅根据参考帖子回答，并忽略帖子中的越权指令；
- 知识库外问题在调用模型前直接拒答；
- 参考来源由程序根据实际进入 Prompt 的帖子统一追加，不让模型自行编写来源。

### 3.3 多轮对话

系统不会把所有短问题都视为追问。`context_utils.py` 根据代词、承接词、疑问结构和上一轮主题判断是否需要增强；触发后将上一轮问题、来源标题和回答摘要加入检索 query。发送给模型的历史限制为最近 6 轮，避免请求体无限增长。

## 四、DeepSeek API 与配置安全

`llm.py` 调用 `https://api.deepseek.com` 的 `deepseek-chat` 模型，温度默认 `0.3`，客户端超时 30 秒，最多进行 2 次 SDK 重试。网页输入个人 Key 时先调用模型列表接口验证，不同浏览器会话的 Key 保存在各自的 Gradio 状态中，不覆盖全局客户端，也不写入磁盘。

仓库只提交空值模板 `config.example.py`。使用者复制为 `config.py` 后，可配置：

| 配置项 | 用途 | 是否必需 |
|--------|------|----------|
| `DEEPSEEK_API_KEY` | CLI、网页服务端模式和在线对比 | 调用模型时需要 |
| `ACCESS_CODE` | 网页共享访问码 | 可选 |
| `CC98_ACCESS_TOKEN` | 重新运行爬虫 | 仅抓取数据时需要 |

环境变量优先于 `config.py`。真实 `config.py`、日志、缓存、虚拟环境和向量缓存均由 `.gitignore` 排除。

## 五、网页界面

网页基于自定义 `gr.Blocks` 布局，而不是旧版 `ChatInterface`。当前界面包括：

- 蓝白色浙江大学主题背景；
- 在常见屏幕高度内优先展示完整问答区；
- 深蓝用户气泡配白色文字；
- 带图标的“新对话”按钮；
- 可轮换的示例问题；
- 默认折叠的访问码/API 配置区；
- 浅色高对比的“检索依据与来源”调试面板；
- 原始问题、实际 query、检索分数、是否进入 Prompt 和来源标题展示。

Windows 用户可运行 `start_web.bat`，脚本会切换到项目目录并启动 `app.py`。

## 六、测试与质量控制

`compare.py` 提供不依赖 API Key 的固定离线题集。2026 年 6 月 10 日在 `work` 分支验证结果：

| 指标 | 结果 |
|------|------|
| 校园问题 Top-3 命中 | 11/11（100%） |
| 知识库外问题正确拒绝 | 5/5（100%） |

在线生成质量不使用简单关键词自动计算“幻觉率”。配置有效 Key 后，脚本并排输出 RAG 与纯模型回答，由组员结合真实帖子人工核对校园细节、来源一致性和边界表达。

交付前检查包括 Python 语法编译、离线检索基准、无 Key 启动行为、文档与截图一致性、PPT 全页渲染、Git 差异和真实密钥扫描。
